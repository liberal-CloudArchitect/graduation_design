# UI设计规格文档

> **产品名称**: 文献分析大数据平台 (LiterAI)  
> **版本**: v1.0  
> **设计系统**: Ant Design 5.x  
> **日期**: 2026年1月

---

## 一、设计原则

### 1.1 核心原则

| 原则 | 描述 | 实践 |
|-----|------|------|
| **清晰** | 信息层级分明，易于理解 | 合理使用留白、字体大小 |
| **高效** | 减少用户操作成本 | 快捷键、默认值、批量操作 |
| **可靠** | 操作有反馈，结果可预期 | 加载状态、成功/失败提示 |
| **专业** | 符合学术工具定位 | 简洁配色、专业排版 |

### 1.2 设计目标

- **首次使用**: 5分钟内完成首次文献上传和问答
- **学习成本**: 无需阅读帮助文档即可使用核心功能
- **效率提升**: 比传统阅读方式节省50%时间

## 六、交互规范

### 6.1 加载状态

| 场景 | 加载方式 | 时长阈值 |
|-----|---------|---------|
| 页面切换 | 顶部进度条 | >200ms 显示 |
| 按钮操作 | 按钮内Spinner | 立即显示 |
| 列表加载 | 骨架屏 | >500ms 显示 |
| 问答生成 | 打字机效果 | 流式显示 |

### 6.2 反馈机制

| 操作 | 反馈类型 | 示例 |
|-----|---------|------|
| 上传成功 | Message成功 | "文献上传成功" |
| 操作失败 | Message错误 | "上传失败，请重试" |
| 删除确认 | Modal确认 | "确定删除这篇文献吗？" |
| 表单验证 | 行内错误 | "请输入有效的邮箱地址" |

### 6.3 快捷键

| 快捷键 | 功能 |
|-------|------|
| `Ctrl/Cmd + K` | 全局搜索 |
| `Ctrl/Cmd + N` | 新建对话 |
| `Ctrl/Cmd + U` | 上传文献 |
| `Enter` | 发送问题 |
| `Shift + Enter` | 换行 |
| `Esc` | 关闭弹窗 |
