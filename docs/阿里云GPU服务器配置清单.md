# LiterAI 项目 — 阿里云 GPU 云服务器配置清单

> **用途**: 生产环境主力机，承载 MinerU 推理、BGE-M3/Reranker、全量 Docker 服务（PostgreSQL / MongoDB / Redis / Milvus / Elasticsearch / Neo4j）及 FastAPI 后端。  
> **配合**: Mac Mini M4 作为开发机，通过公网或 VPN 访问本 ECS API。  
> **说明**: 阿里云不提供消费级 RTX 4090，采用 **gn7i 规格族（NVIDIA A10 24GB）** 作为等价方案，显存与算力可满足本项目需求。

---

## 一、实例规格（ECS 选型）

### 1.1 规格族说明

| 项目 | 说明 |
|-----|------|
| **规格族** | **GPU 计算型 gn7i**（NVIDIA A10 GPU，24 GB 显存） |
| **等价关系** | 原「桌面级 4090 24GB」→ 阿里云 **gn7i（A10 24GB）**，显存与 CUDA 兼容性满足 MinerU + BGE-M3 + Reranker |
| **镜像类型** | 仅支持 **I/O 优化实例**，系统盘/数据盘需选 **ESSD 云盘** |

### 1.2 推荐规格（按预算选一档）

| 规格名称 | vCPU | 内存 (GiB) | GPU | 显存 | 适用场景 | 约价参考（按量） |
|---------|------|------------|-----|------|---------|------------------|
| **ecs.gn7i-c16g1.4xlarge** | 16 | 60 | A10 × 1 | 24 GB | **推荐**：兼顾成本与多容器 | ~1.9 元/时 |
| ecs.gn7i-c8g1.2xlarge | 8 | 30 | A10 × 1 | 24 GB | 预算紧、轻量部署 | 较低 |
| ecs.gn7i-c32g1.8xlarge | 32 | 188 | A10 × 1 | 24 GB | 高并发、大批量预处理 | 较高 |

**推荐首选**：**ecs.gn7i-c16g1.4xlarge**（16 vCPU + 60 GiB 内存 + A10 24GB），与方案中的「64 GB 内存」接近，跑满 Docker 全家桶 + 模型无压力。

### 1.3 显存与内存预算（与方案一致）

```
GPU 显存 (24 GB, A10):
  MinerU (Qwen2VL-1.2B)     4–6 GB   (推理时)
  BGE-M3 (dense+sparse)    ~2 GB
  BGE-Reranker-v2-m3      ~1 GB
  ─────────────────────────────────
  合计                     7–9 GB   (可同时常驻)

实例内存 (60 GiB, 以 4xlarge 为例):
  PostgreSQL / MongoDB / Redis / Milvus / ES / Neo4j / Backend + 系统
  合计约 20–35 GiB，60 GiB 充裕。
```

## 二、存储配置（系统盘 + 数据盘）

### 2.1 系统盘

| 项目 | 建议 |
|-----|------|
| **类型** | **ESSD 云盘**（gn7i 仅支持 ESSD） |
| **容量** | **100 GB**（最小 20 GB，建议 ≥80 GB 以装系统、Docker、Python、少量模型） |
| **性能** | ESSD PL0 即可；若需更高 IOPS 可选 PL1 |

系统盘在**创建 ECS 时**选定，不可后补，建议一次选够。

### 2.2 数据盘（模型 + 数据库持久化）

| 项目 | 建议 |
|-----|------|
| **类型** | **ESSD 云盘** |
| **容量** | **200–500 GB**（模型约 6 GB + 代码与日志 10 GB + 数据库随使用增长，建议预留 100 GB+） |
| **挂载** | 创建实例时**随实例创建并挂载**，或实例创建后**单独创建云盘再挂载** |
| **挂载点** | 建议挂载到 `/data`，并在其下建 `models`、`graduation`、`persistence` 等目录 |

数据盘创建后需在实例内**分区、格式化、挂载**（若选「挂载到实例」且未选「初始化」，则需自行执行 `fdisk` + `mkfs.ext4` + `mount` 并写入 `/etc/fstab`）。

### 2.3 磁盘占用粗算

| 内容 | 预估大小 |
|-----|----------|
| 系统 + Docker + Python | ~20–30 GB（放系统盘） |
| MinerU + BGE-M3 + BGE-Reranker | ~6 GB（放数据盘 /data/models） |
| 代码与日志 | ~5–10 GB |
| 数据库与向量库 | 预留 50–100 GB |
| **数据盘建议** | **≥ 200 GB**，宽裕选 **500 GB** |

## 三、镜像与驱动

### 3.1 镜像选择

| 优先级 | 镜像类型 | 说明 |
|-------|---------|------|
| **1** | **云市场「GPU 镜像」** | 搜索「Ubuntu 22.04 GPU」或「NVIDIA 驱动 GPU」，选带 **NVIDIA 驱动 + CUDA 12.x** 的镜像，省去自装驱动 |
| 2 | **阿里云官方 Ubuntu 22.04** | 若未选 GPU 镜像，需在实例内**自行安装** NVIDIA 驱动与 CUDA（通过 runfile 或 apt） |

创建实例时在「镜像」中选择 **「镜像市场」** 并搜索 GPU 相关镜像，或选 **「公共镜像」→ Ubuntu 22.04 64位** 再手动装驱动。

### 3.2 驱动与 CUDA 版本

| 项目 | 要求 |
|-----|------|
| **NVIDIA 驱动** | 535.x 或 545.x+（支持 A10） |
| **CUDA** | 12.1 或 12.4（与 PyTorch 2.x / Transformers 兼容） |

创建并登录实例后执行：

```bash
nvidia-smi
# 应看到 NVIDIA A10、24GB 显存、驱动版本
```

若未预装，需在实例内安装驱动与 CUDA（参考阿里云文档「安装 NVIDIA 驱动」）。

## 四、网络与安全组

### 4.1 专有网络 VPC

- 使用默认 VPC 或新建 VPC 均可。
- 若 Mac Mini 通过**公网**访问 ECS，需为该 ECS 分配**公网 IP**（创建时选「分配公网 IPv4」或后绑 EIP）。

### 4.2 安全组规则（入方向）

在 ECS 所属安全组中放行以下端口（仅开放业务与运维所需端口）：

| 端口 | 协议 | 授权对象 | 说明 |
|-----|------|---------|------|
| 22 | TCP | 本机公网 IP 或 0.0.0.0/0（建议收紧为固定 IP） | SSH |
| 8000 | TCP | 0.0.0.0/0 或指定 IP | Backend API（或通过 Nginx 只开 80/443） |
| 80 | TCP | 0.0.0.0/0 | HTTP（Nginx） |
| 443 | TCP | 0.0.0.0/0 | HTTPS（可选） |

**说明**：PostgreSQL / Redis / MongoDB / Milvus / ES / Neo4j 仅需本机或内网访问时，**不必**在安全组开放其端口，只开放 22 与 8000（或 80/443）即可。若需从本机直连数据库调试，可临时加规则或通过 SSH 隧道。

### 4.3 带宽

- **按固定带宽**：建议 **5 Mbps** 起（对外提供 API 或拉取模型时更稳）。
- **按使用流量**：适合访问量不大、偶发流量，注意控制费用。

## 五、创建 ECS 时的勾选汇总（控制台选项）

在阿里云 ECS 控制台「创建实例」时，按下面逐项选择即可对应本方案：

| 配置项 | 选择 |
|-------|------|
| **地域与可用区** | 选离自己/用户近的（如华东1、华北2） |
| **实例规格** | **GPU 计算型** → **gn7i** → **ecs.gn7i-c16g1.4xlarge**（或 c8g1.2xlarge / c32g1.8xlarge） |
| **镜像** | **镜像市场** → 带 **NVIDIA 驱动 + CUDA 12** 的 **Ubuntu 22.04**；若无则选 **Ubuntu 22.04 64位** 后自装驱动 |
| **存储** | 系统盘：**ESSD 云盘 100 GB**；数据盘：**ESSD 云盘 200–500 GB**，挂载到该实例 |
| **网络** | 默认 VPC 或自建 VPC；若需公网访问勾选 **分配公网 IPv4**，带宽选 **5 Mbps** 或按量 |
| **安全组** | 新建或已有安全组，入方向放行 **22、8000、80、443**（见上表） |
| **登录凭证** | **密钥对**（推荐）或密码，用于 SSH |

创建完成后，在实例内挂载并格式化数据盘（若未自动挂载），再安装 Docker、Python、拉代码、部署服务。

## 六、软件栈与目录（与通用清单一致）

### 6.1 基础环境

| 软件 | 版本建议 |
|-----|----------|
| Python | 3.10 或 3.11 |
| Docker | 24.x+ |
| Docker Compose | v2.x |
| Git | 2.x |

### 6.2 项目依赖服务（Docker）

与 `docker-compose.yml` 及 `backend/app/core/config.py` 一致：

| 服务 | 端口 (宿主机) |
|-----|----------------|
| PostgreSQL | 5432 |
| MongoDB | 27017 |
| Redis | 6379 |
| Milvus | 19530, 9091 |
| Elasticsearch | 9200 或 9201 |
| Neo4j | 7474, 7687 |
| Backend (uvicorn) | 8000 |

### 6.3 目录建议（数据盘挂载到 /data 时）

```text
/data/
├── models/
│   ├── MinerU2.5-2509-1.2B/
│   ├── bge-m3/
│   └── bge-reranker-v2-m3/
├── graduation/          # git clone 项目
└── persistence/          # Docker 数据卷映射（可选）
```

## 七、计费方式建议

| 方式 | 适用场景 |
|-----|---------|
| **按量付费** | 短期开发、评测，用完即停，按小时计费 |
| **包月** | 稳定跑 1 个月以上，通常比按量省 15%–30% |

gn7i 按量约 **1.9 元/小时** 起（以 c16g1.4xlarge 为例），包月价以控制台为准。数据盘单独按容量计费（ESSD 按 GB/月）。

## 八、验收检查清单（实例创建并部署后）

- [ ] `nvidia-smi` 可见 **NVIDIA A10、24 GB**、驱动版本正常。
- [ ] `python3 --version` 为 3.10 或 3.11。
- [ ] `docker --version`、`docker compose version` 可用。
- [ ] 数据盘已挂载到 `/data`（或自定路径），`df -h` 可见。
- [ ] `docker compose up -d` 能正常启动 Postgres / Mongo / Redis / Milvus / ES / Neo4j。
- [ ] 后端能连上上述服务；MinerU、BGE-M3、BGE-Reranker 可加载且推理无 CUDA 报错。
- [ ] 从 Mac Mini 能通过 **公网 IP:8000**（或 80/443）访问 Backend API；安全组已放行对应端口。

## 九、与通用「4090 配置清单」的对应关系

| 通用清单（桌面级） | 阿里云 ECS 对应 |
|------------------|----------------|
| RTX 4090 24 GB | **gn7i（NVIDIA A10 24 GB）** |
| 12 核 64 GB 内存 | **ecs.gn7i-c16g1.4xlarge（16 vCPU, 60 GiB）** |
| 系统盘 256 GB NVMe | **系统盘 ESSD 100 GB** |
| 数据盘 512 GB SSD | **数据盘 ESSD 200–500 GB** |
| 自装 Ubuntu + 驱动 | **镜像市场 GPU 镜像** 或 **Ubuntu 22.04 + 自装驱动** |

其余软件栈、端口、Docker Compose、Neo4j 增补、后端配置与 **docs/4090服务器租用配置清单.md** 中第三至六节一致，可直接复用。
