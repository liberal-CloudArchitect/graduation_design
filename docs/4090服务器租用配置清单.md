# LiterAI 项目 — 4090 服务器租用完整配置清单

> **用途**: 生产环境主力机，承载 MinerU 推理、BGE-M3/Reranker、全量 Docker 服务（PostgreSQL / MongoDB / Redis / Milvus / Elasticsearch / Neo4j）及 FastAPI 后端。  
> **配合**: Mac Mini M4 作为开发机，通过 HTTP 调用本机 API 或直接连本机服务调试。

---

## 一、硬件配置（租机必选）

### 1.1 最低推荐配置

| 项目 | 规格 | 说明 |
|-----|------|------|
| **GPU** | **NVIDIA RTX 4090 × 1** | 24 GB GDDR6X，必须项 |
| **CPU** | 12 核 / 24 线程及以上（如 Intel i7-12700 / AMD Ryzen 9 5900X） | 跑 Docker 多容器 + 预处理 |
| **内存** | **64 GB DDR4/DDR5** | 推荐 64GB：ES/Milvus/Neo4j/Postgres 同时跑 + 预留 |
| **系统盘** | **NVMe SSD 256 GB+** | 装系统、Docker、Python、少量模型 |
| **数据盘** | **NVMe SSD 512 GB+** 或 **SATA SSD 1 TB+** | 模型权重、向量库与业务数据持久化 |

### 1.2 显存与内存预算（供选机参考）

```
GPU 显存 (24 GB):
  MinerU (Qwen2VL-1.2B)     4–6 GB   (推理时)
  BGE-M3 (dense+sparse)    ~2 GB
  BGE-Reranker-v2-m3      ~1 GB
  ─────────────────────────────────
  合计                     7–9 GB   (可同时常驻，余量充足)

系统内存 (建议 64 GB):
  PostgreSQL                2–4 GB
  MongoDB                   2–4 GB
  Redis                     0.5–1 GB
  Milvus (standalone)       2–4 GB
  Elasticsearch (单节点)     2–4 GB
  Neo4j (社区版)            1–2 GB
  Backend (uvicorn)         1–2 GB
  系统 + 缓冲               8–12 GB
  ─────────────────────────────────
  合计                      ~20–35 GB  (64 GB 充裕)
```

### 1.3 若预算有限（降档方案）

| 项目 | 最低可接受 | 影响 |
|-----|-----------|------|
| 内存 | 32 GB | 需降低 ES/Milvus 的 JVM/进程内存，或减少并发容器 |
| 数据盘 | 256 GB SSD | 需控制模型数量或只放 BGE-M3 + Reranker，MinerU 按需加载 |
| CPU | 8 核 16 线程 | 多容器同时跑时 CPU 略紧，PDF 解析排队略多 |

**GPU 不可降档**：必须 RTX 4090（24 GB），否则 MinerU + BGE-M3 + Reranker 无法同时常驻。

## 二、操作系统与驱动

### 2.1 系统

| 项目 | 要求 |
|-----|------|
| **操作系统** | **Ubuntu 22.04 LTS (64-bit)** 推荐；Ubuntu 20.04 亦可 |
| **内核** | 5.x 及以上（便于新驱动与 Docker） |
| **权限** | 建议 **root 或 sudo**，便于装驱动、Docker、挂盘 |

### 2.2 NVIDIA 驱动与 CUDA

| 项目 | 版本/说明 |
|-----|----------|
| **NVIDIA 驱动** | **535.x 或 545.x+**（支持 RTX 4090） |
| **CUDA** | **12.1 或 12.4**（与 PyTorch 2.x / Transformers 兼容） |
| **cuDNN** | 与 CUDA 版本匹配（若用官方 runfile 安装驱动，可选装 CUDA Toolkit 自带） |

安装后需满足：

```bash
nvidia-smi
# 能看到 RTX 4090、24GB 显存、驱动版本
```

若租用平台已预装「GPU 镜像」（如 Ubuntu + 驱动 + CUDA），优先选该镜像，可省去自行装驱动步骤。

## 三、软件栈与运行环境

### 3.1 基础环境

| 软件 | 版本建议 | 用途 |
|-----|----------|------|
| **Python** | **3.10 或 3.11** | 与当前 `requirements.txt` 一致 |
| **pip** | 最新 | 安装依赖 |
| **Docker** | 24.x+ | 跑所有中间件与可选后端 |
| **Docker Compose** | v2.x | 编排多容器 |
| **Git** | 2.x | 拉取代码与模型路径配置 |

### 3.2 项目依赖的服务（Docker 部署）

以下端口与当前项目 `backend/app/core/config.py` 及现有 `docker-compose.yml` 对齐：

| 服务 | 镜像/版本 | 端口 (宿主机) | 说明 |
|-----|-----------|----------------|------|
| **PostgreSQL** | postgres:16 | 5432 | 业务库 |
| **MongoDB** | mongo:7 | 27017 | 文献分块与缓存 |
| **Redis** | redis:7-alpine | 6379 | 缓存与队列 |
| **Milvus** | milvusdb/milvus:v2.3.4 (standalone) | 19530, 9091 | 向量检索（依赖 etcd + minio） |
| **Elasticsearch** | elasticsearch:8.12.0 | 9200 | BM25 全文检索 |
| **Neo4j** | neo4j:5 (Community) | 7474(HTTP), 7687(Bolt) | GraphRAG（阶段 6） |

若 ES 需与现有开发环境一致，可保留 `9201:9200` 映射，后端通过 `ES_HOST=localhost`、`ES_PORT=9201` 连接。

### 3.3 后端与模型（宿主机直跑）

| 项目 | 说明 |
|-----|------|
| **Backend** | FastAPI 应用在宿主机用 `uvicorn` 跑，便于挂载大模型路径、调试和重启；也可改为 Docker 运行，需挂载模型卷。 |
| **MinerU** | 使用工作区 `MinerU2.5-2509-1.2B/`，需上传到服务器或通过挂载/同步到固定目录，如 `/data/models/MinerU2.5-2509-1.2B`。 |
| **BGE-M3** | 可从 HuggingFace 下载到 `/data/models/bge-m3` 或使用项目已有 `bge-m3/`。 |
| **BGE-Reranker-v2-m3** | 建议目录 `/data/models/bge-reranker-v2-m3`。 |

## 四、目录与存储规划

### 4.1 建议目录结构

```text
/data/

│   └── ...
```

| 内容 | 预估大小 |

| MinerU2.5-2509-1.2B | ~2.5 GB |

| BGE-Reranker-v2-m3 | ~1 GB 级 |

| **合计** | **建议数据盘 ≥ 512 GB** |

## 五、网络与端口

### 5.1 宿主机需开放端口（供本机或 Mac Mini 访问）

| 端口 | 服务 | 说明 |
|-----|------|------|
| 22 | SSH | 运维与部署 |
| 5432 | PostgreSQL | 后端连接 |
| 6379 | Redis | 后端连接 |
| 27017 | MongoDB | 后端连接 |
| 9200 或 9201 | Elasticsearch | 后端连接（与配置一致） |
| 19530, 9091 | Milvus | 后端连接 |
| 7474, 7687 | Neo4j | 阶段 6 使用 |
| 8000 | Backend API | FastAPI |
| 80 / 443 | Nginx（若部署） | 对外 HTTP/HTTPS |

若仅 Mac Mini 访问，可只对内网开放；若需公网访问，建议 80/443 经 Nginx 反向代理，并配置防火墙与密钥。

### 5.2 后端连接配置（.env 或环境变量）

在 4090 服务器上运行后端时，建议使用本机地址连接各服务（若 Docker 跑在宿主机）：

```bash
POSTGRES_HOST=localhost
POSTGRES_PORT=5432
MONGODB_URI=mongodb://localhost:27017/
REDIS_URL=redis://localhost:6379/0
MILVUS_HOST=localhost
MILVUS_PORT=19530
ES_HOST=localhost
ES_PORT=9200
# 若 ES 映射为 9201，则 ES_PORT=9201
NEO4J_URI=bolt://localhost:7687
```

若后端也容器化，则上述 host 改为 Docker 服务名（如 `postgres`、`milvus`、`elasticsearch`、`neo4j`）。

## 六、Docker Compose 增补（Neo4j）

在现有 `docker-compose.yml` 上增加 Neo4j 服务（阶段 6 使用），例如：

```yaml
  neo4j:
    image: neo4j:5-community
    container_name: graduation_neo4j
    restart: unless-stopped
    environment:
      NEO4J_AUTH: neo4j/your_password_here
    ports:
      - "7474:7474"
      - "7687:7687"
    volumes:
      - neo4j_data:/data
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:7474"]
      interval: 10s
      timeout: 5s
      retries: 5

volumes:
  # ... 现有 volumes ...
  neo4j_data:
```

生产环境请将 `your_password_here` 改为强密码，并在后端配置中写入 `NEO4J_URI` 与认证信息。
