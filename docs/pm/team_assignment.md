## 四、协作机制

### 4.1 会议机制

| 会议类型 | 频率 | 参与者 | 时长 | 目的 |
|---------|-----|-------|------|-----|
| 站会 | 每日 | 全员 | 15min | 同步进展、暴露阻塞 |
| 周会 | 每周 | 全员 | 1h | 回顾总结、计划下周 |
| 评审会 | 每两周 | 全员+导师 | 2h | 里程碑验收 |
| 技术讨论 | 按需 | 相关人员 | 1h | 解决技术问题 |

### 4.2 沟通工具

| 工具 | 用途 |
|-----|------|
| 飞书/钉钉 | 日常沟通、文档协作 |
| Git | 代码版本管理 |
| GitHub Issues | 任务跟踪 |
| Figma | 设计协作 |

### 4.3 文档规范

- **存放位置**: `docs/` 目录
- **命名规范**: `[类型]_[名称]_[版本].md`
- **评审流程**: 提交 → 评审 → 修改 → 归档

## 五、考核标准

| 角色 | 考核指标 |
|-----|---------|
| PM | 里程碑按时完成率≥90%、风险及时处理 |
| PD | 需求变更率≤10%、用户满意度≥4.0 |
| 后端Lead | API完成率100%、测试覆盖率≥80% |
| 前端Lead | 核心页面完成100%、无P0级Bug |
| AI工程师 | RAG准确率≥85%、模型按期交付 |
