# 文献分析大数据平台 - 最终可行性研究报告

> **版本**: 综合版 v3.0 (Agent记忆增强版)  
> **日期**: 2026年2月  
> **方案定位**: RAG-First + 微调增强 + **Agent记忆系统**

---

## 📋 报告摘要

本报告综合了"大模型微调方案"与"RAG+Agent架构方案"的优势，提出**"RAG-First + 微调增强 + Agent记忆"**架构，实现：

- **RAG优先**：MVP阶段即包含RAG问答功能，不在传统NLP上浪费时间
- **小模型专精**：LayoutLMv3布局分析、BERT关键词NER
- **按需微调**：BGE-M3先基线评测，Recall@10>85%则跳过微调
- **端云协同**：本地模型保护隐私，云端API提供强大能力
- **🆕 Agent记忆系统**：动态记忆 + 交叉记忆 + 重构记忆，实现类人脑的智能记忆管理

> **版本历史**:
> - **v3.0 (2026年2月)**：新增 **Agent记忆系统**（动态/交叉/重构三大记忆模式）
> - v2.0 (2026年1月)：PDF解析改用LayoutLMv3，BGE先基线后微调，RAG前置到Phase 2
> - v1.0 (2025年12月)：初始版本

## 第一章 项目概述

### 1.1 项目背景

学术文献数量呈爆发式增长，科研人员面临三大痛点：
1. **信息过载**：海量文献难以高效筛选
2. **分析困难**：缺乏智能化工具辅助提取关键信息
3. **写作低效**：文献综述、趋势分析耗时耗力

### 1.2 项目目标

构建一个**"从阅读到写作"全流程智能辅助**的学术文献分析平台：

| 目标 | 描述 |
|-----|------|
| 智能解析 | 精准提取PDF文献结构化信息 |
| RAG问答 | 基于文献的自然语言问答，带引用溯源 |
| 趋势分析 | 研究热点识别、突现词检测、发展预测 |
| 写作辅助 | 论文大纲生成、文献综述、段落润色 |
| 开放集成 | 连接Semantic Scholar等1.25亿+学术文献 |

### 2.2 核心技术选型

#### 2.2.1 微调模型矩阵

| 功能任务 | 微调模型 | 微调策略 | 训练数据 | 性能提升 |
|---------|---------|---------|---------|---------|
| **PDF解析** | ChatGLM3-6B | LoRA (r=8) | 5,000篇标注论文 | +20% F1 |
| **向量化** | BGE-M3 | 对比学习 | 50,000摘要对 | +15% Recall |
| **关键词** | BERT-base | 序列标注 | 10,000篇BIO标注 | +21% F1 |

#### 2.2.2 RAG技术栈

| 组件 | 技术选型 | 用途 |
|-----|---------|------|
| 框架 | LangChain + LlamaIndex | RAG流程编排 |
| 检索 | BM25 + Milvus + BGE | 三路召回混合检索 |
| 重排 | BGE-Reranker | 提升检索精度 |
| 生成 | GPT-4o / DeepSeek-V3 / ChatGLM4 | 多模型路由 |

#### 2.2.3 外部API集成

| API | 文献量 | 用途 | 免费额度 |
|-----|-------|------|---------|
| Semantic Scholar | 2亿+ | 元数据+引用 | 100次/秒 |
| OpenAlex | 2.5亿+ | 开放数据 | 无限制 |
| arXiv | 200万+ | 预印本 | 无限制 |
| CrossRef | 1.3亿+ | DOI解析 | 50次/秒 |

#### 2.2.4 🆕 Agent记忆系统 (v3.0新增)

| 模块 | 技术实现 | 功能 |
|------|----------|------|
| **动态记忆** | Milvus + BGE-M3 | 交互记录的写入/检索/更新 |
| **交叉记忆** | Milvus JSON字段 | Agent间记忆共享与权限控制 |
| **重构记忆** | LLM + Cue提取 | 基于线索的生成式记忆重构 |
| **快慢思考** | 规则分类器 | System 1(快)/System 2(慢)分流 |
| **异步反思** | asyncio队列 | 后台Worker处理记忆整合 |
| **遗忘机制** | 衰减函数+保护期 | 防止数据库膨胀 |

> **技术选型决策**: 采用 **Milvus metadata** 模拟轻量级图谱关系，避免引入Neo4j带来的额外复杂度。当前需求仅需1跳关系查询，Milvus完全满足。
