# 项目管理与产品实施计划

> **角色**: 项目经理(PM) + 产品经理(PD)  
> **职责**: PM管进度资源，PD管需求设计

---

## 10个月关键节点

| 月份 | PM任务 | PD任务 |
|-----|-------|-------|
| M1 | 团队组建、章程制定 | 用户调研、PRD撰写 |
| M2 | Sprint规划 | 原型设计 |
| M3-M4 | 进度监控 | MVP验收 |
| M5-M6 | 资源协调(GPU) | 微调验收标准 |
| M7-M8 | 风险管理 | Agent验收 |
| M9 | 测试协调 | UAT测试 |
| M10 | 上线交付 | 运营准备 |

---

## PM核心工作

### Sprint机制
- 周期: 2周/Sprint
- 每日站会: 15min
- Sprint评审: 每2周

### 风险清单

| 风险 | 缓解措施 |
|-----|---------|
| 微调效果差 | 基线评测+按需微调 |
| GPU资源不足 | 云服务备选 |
| 进度延期 | 及时预警+优先级调整 |

---

## PD核心工作

### PRD结构
1. 产品概述
2. 功能需求(MVP + 高级)
3. 非功能需求
4. 验收标准

### 验收标准

| 阶段 | 标准 |
|-----|------|
| M5 MVP | RAG问答可用，响应≤5s |
| M7 微调 | 模型F1≥85% |
| M9 Agent | 综述生成完整 |

---

## 交付物

- PM: 项目章程、进度报告、项目总结
- PD: PRD、原型、验收报告、用户手册
