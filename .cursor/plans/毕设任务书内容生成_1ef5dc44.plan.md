**写作策略**: 参照范例（约 1000 字），从宏观背景到具体课题逐步收窄，包含以下逻辑层次：

2. **现有工具的不足** — Zotero/EndNote 缺乏智能分析、ChatGPT 存在幻觉且无法处理私有文献、Semantic Scholar 功能分散

5. **工程意义** — 对智能制造工程专业学生的综合能力锻炼（全栈开发 + AI + 大数据 + 系统设计），符合学科交叉创新方向
6. **应用价值** — 提升文献阅读效率、辅助综述写作、支持多人协作科研

## 第 2 部分：毕业设计（论文）任务的内容和要求

**写作策略**: 参照范例的 5-6 条结构化任务项，覆盖文献调研到论文撰写全流程：

1. 文献调研与开题 — 调研 RAG、向量检索、大语言模型、PDF 智能解析等领域的国内外研究现状，完成开题报告和外文文献翻译
2. 系统架构设计 — 完成前后端分离架构设计（React + FastAPI），设计多存储互补方案（PostgreSQL + MongoDB + Milvus + Elasticsearch + Redis），绘制系统架构图和数据库 ER 图
3. RAG 核心引擎开发 — 实现 PDF 解析（MinerU/PDFPlumber）、语义分块（父子文档索引）、BGE-M3 向量化、BM25+Dense+RRF 混合检索、重排序、LLM 流式生成与引用溯源
4. 平台功能开发 — 完成文献管理、智能问答、知识图谱可视化、趋势分析、写作辅助、多 Agent 协作编排等功能模块的开发与集成
5. 系统测试与评估 — 建立评测基线，对解析质量、检索召回率（Recall@K）、答案准确率、引用准确率等指标进行量化评测
6. 论文撰写 — 将设计方案、实现细节、测试结果整理成毕业设计论文
