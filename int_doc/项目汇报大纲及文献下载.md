## 📑 PPT结构大纲

### 第一章: 研究背景与课题意义 (3-4页)

#### 1.1 研究背景
- 学术文献数量爆发式增长 (年增长率超过5%)
- 研究生文献阅读面临的三大痛点:
  - 单篇文献逻辑结构复杂，核心论点难以快速把握
  - 领域文献数量呈指数级增长，手动梳理效率极低
  - 高频词汇与研究热点统计耗时且易遗漏

#### 1.2 课题意义
- 与课程需求的对应关系:
  | 课程要求 | 本项目实现 |
  |---------|-----------|
  | 单篇文献逻辑图生成 | LayoutLMv3 + LLM信息整合 |
  | 领域文献思维导图 | 知识图谱 + Multi-Agent协作 |
  | 高频词汇汇总 | TF-IDF + TextRank + BERT NER |
  | 研究热点分析 | RAG问答 + 趋势预测 |

#### 1.3 项目目标
- 构建"从阅读到写作"全流程智能辅助平台
- 实现RAG增强的文献分析与问答能力

#### 2.2 核心技术选型
| 技术组件 | 选型方案 | 技术优势 |
|---------|---------|---------|
| 向量嵌入 | BGE-M3 | 多语言+多功能+多粒度 |
| PDF解析 | LayoutLMv3 | 专业布局分析,仅400MB |
| 关键词提取 | BERT + TF-IDF | 序列标注+统计方法结合 |
| RAG框架 | LangChain + LlamaIndex | 成熟的RAG编排能力 |
| 向量数据库 | Milvus | 高性能混合检索 |

#### 2.3 RAG-First开发策略
- **核心创新**: MVP阶段即包含RAG问答功能
- **技术决策**: 
  - LayoutLMv3替代ChatGLM进行PDF解析 (更专业、更轻量)
  - BGE-M3先基线评测,按需微调 (防止灾难性遗忘)

### 第三章: 核心功能实现 (6-8页)

#### 3.1 PDF文献解析 (对应课程: 单篇文献逻辑分析)
- 多层解析架构:
  1. PDFPlumber文本提取
  2. Tesseract OCR (扫描件)
  3. LayoutLMv3布局分析 (识别标题、摘要、章节)
  4. LLM信息整合 (元数据提取)

#### 3.2 RAG智能问答 (对应课程: 检索增强生成)
- 混合检索策略: BM25 + 向量检索 + 稀疏检索
- 重排序: BGE-Reranker
- 引用溯源: 答案关联原文片段

#### 3.3 关键词提取与高频词分析
- TF-IDF权重计算
- TextRank图算法
- BERT序列标注 (关键词NER)
- 语义归一化处理

#### 3.4 知识图谱与趋势分析
- 文献引用网络构建
- 研究热点识别
- 时间序列趋势预测

#### 3.5 Multi-Agent协作系统
- 检索Agent: 文献召回
- 分析Agent: 趋势分析
- 写作Agent: 综述生成
- 搜索Agent: 外部API集成
